Prüfe die UI/UX kritisch:

1. Lies relevante UI-Dateien (Screens, Widgets, Styles).
2. Prüfe (Priorität: mobile first):
   - **Visuelle Hierarchie:** Ist das Wichtigste sofort erkennbar?
   - **Bedienbarkeit:** Sind Tap-Targets groß genug? Navigation klar?
   - **Konsistenz:** Stimmen Abstände, Farben, Typografie mit dem Rest überein?
   - **Edge Cases:** Leere Zustände, lange Texte, kleine Screens?
3. Schwächen priorisiert auflisten (kritisch / mittel / niedrig).
4. Konkrete Korrekturvorschläge – kein Schönreden.

**Verboten:** Keine Umsetzung ohne Plan, kein Commit ohne Auftrag.
