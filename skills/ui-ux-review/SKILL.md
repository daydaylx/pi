# UI-/UX-Review

Bewerte die vorhandene Oberfläche anhand der verfügbaren Artefakte, etwa Code, Screenshots oder laufender Anwendung. Respektiere `AGENTS.md` und die aktive Permission-Policy.

- Benenne die folgenreichsten Bedienprobleme statt bloßer Geschmacksfragen.
- Prüfe Informationshierarchie, visuelle Dominanz, Layout, Farben, Typografie, Zustände und Konsistenz.
- Begründe jede Empfehlung anhand des Nutzerflusses oder eines konkreten Befunds und priorisiere sie.
- Ändere keine UI ohne ausdrückliche Freigabe.

Strukturiere das Ergebnis mit:

## Größte Schwächen
## Bedienprobleme
## Visuelle Dominanz
## Inkonsistenzen
## Konkrete Verbesserungen
## Priorität
