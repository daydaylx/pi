# Release / Changelog

Sammle Änderungen seit dem vom Nutzer benannten oder sinnvoll ermittelbaren Vergleichspunkt. Respektiere `AGENTS.md` und die aktive Permission-Policy.

- Ermittle letzten Release-Tag, Versionsstand oder einen klar genannten Baseline-Commit; nenne die Annahme, falls kein Vergleichspunkt existiert.
- Gruppiere die Änderungen nachvollziehbar in Features, Fixes, Breaking Changes und interne Arbeiten.
- Identifiziere Risiken, Upgrade- oder Migrationshinweise und offene Release-Voraussetzungen.
- Erhöhe keine Version, erstelle keinen Tag, kein Release und kein Deployment ohne direkten Nutzerauftrag.

Strukturiere das Ergebnis mit:

## Änderungen
## Fixes
## Risiken
## Breaking Changes
## Migration
## Release-Checkliste
